# 01_overview

## 目的

学童で、子どもが親しめる「マスコットキャラクター」として会話できる存在を作る。

## まず作る形（MVPの設置形態 / 決定）

- 常設PC（Mac mini M4 / 16GB / 256GB）+ モニターにマスコット表示
- ぬいぐるみ筐体は将来の差し替え対象（まずはモニターで成立させる）

## 優先順位（決定）

1. キャラクター一貫性
2. 現場で破綻しない運用（混線しても崩れない）
3. 安全・データ最小化（不要な保存をしない/センシティブを避ける）
4. 後から拡張できる構造（差し替え/追加が容易）

## MVPの段階（決定）

- A: `ROOM`（全体向け雑談 / 個人識別なし / 永続保存なし）
- B0: `PERSONAL(name)`（音声で名乗る / NFCなし / 低センシティブ記憶 + 職員Confirm）
- B: `PERSONAL(child)`（NFC導入で識別を置き換える）
- C: ゲーム（後続。ルールは決定的ロジック、LLMは盛り上げ役に限定）

## MVPで「入れる/入れない」

### 入れる（B0まで）

- Push-to-talk（ボタン押下中だけ認識、待機時は無反応）
- `ROOM` ↔ `PERSONAL(name)` のモード遷移
- 記憶の抽出（ホワイトリスト）→ `pending` 化 → 職員Confirmで `confirmed` 保存
- `confirmed` の想起（`PERSONAL` 中のみ / 低センシティブのみ）
- `ROOM` の「みんなの思い出」（日付単位 / 個人特定なし）

### 入れない（当面）

- バイオメトリクス（声紋など話者認証）
- 録音ファイルの保存
- 会話全文ログの永続保存
- NFCハード依存のフロー（Bで導入）
